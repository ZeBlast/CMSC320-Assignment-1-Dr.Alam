Q1: Why might be consider using git?

Answer: Git may be considered by developers as a version control system for a project or team, allowing them to keep track of code history and helps manage collaboration.

Q3: What is the difference between merging and rebasing?

Answer: Merging a branch "a" into branch "b" would bring the changes from "a" into "b," adding 1 new commit into "b" without affecting the commit history of "a." Rebasing "b" to incorporate the changes of "a" would instead rewrite the commit history by shifting the commits of "b" to occur after the latest commit in "a."
